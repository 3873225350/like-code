# 贪吃蛇游戏 - 多Agent协作规范

## 任务目标
使用 MiniMax-M2.5 和 MiniMax-M2.1 两个Agent共同完成一个贪吃蛇游戏，MiniMax-M2.7 负责监控协调。

## 文件位置
`/home/clashuser/hzh/item_bo/like-code/templete/v3/`

## Agent分工
- **mm25**: 主要开发Agent，负责核心游戏逻辑（蛇移动、碰撞检测、食物生成）
- **mm21**: 辅助开发Agent，负责UI渲染、输入处理、计分系统
- **mm27**: 监控Agent，每10秒检查一次文件变化，最多5次

## 游戏要求
- 简单TUI游戏（终端界面）
- 基础功能：蛇移动、吃食物增长、撞墙/自身死亡、计分
- 无需复杂动画，专注多Agent协作测试

## 验收标准
1. 代码能运行（启动无语法错误）
2. 基础贪吃蛇功能完整
3. 两个Agent代码风格有区分

## 监控规则
mm27监控mm25/mm21：
- 每10秒检查文件变化
- 最多检查5次
- 连续2次无变化时运行语法/启动检查
- 输出最终报告
